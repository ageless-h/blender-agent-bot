# 自定义 Persona 模板说明

本文档用于创建可复用的自定义角色 Prompt，推荐采用 **Base + Model + Persona** 三层继承结构。

## 1. 模板变量

可在模板中使用以下占位符（双大括号语法）：

- `{{model_name}}`：模型名称（如 `gpt-5`、`claude-3.7`）
- `{{blender_version}}`：Blender 版本（如 `4.2`）
- `{{project_type}}`：项目类型（建筑、角色、产品等）
- `{{quality_target}}`：质量目标（草图、预览、最终）
- `{{time_budget}}`：时间预算（30min / 2h / 1day）
- `{{output_format}}`：输出格式（`.blend`、`.glb`、渲染图）

## 2. 三层继承机制

### 2.1 Base 层（通用行为）

定义所有角色共享规则，例如：

- 回答结构（先结论后步骤）
- 错误处理（先定位再修复）
- 安全约束（不执行破坏性操作）

### 2.2 Model 层（模型适配）

根据模型能力覆盖行为，例如：

- 长上下文模型：允许更长计划与分阶段输出
- 快速模型：减少解释长度，强调行动步骤

### 2.3 Persona 层（领域角色）

定义专业能力与工具偏好，例如建筑可视化、角色建模或游戏资产流程。

## 3. 推荐模板骨架

```md
# Persona: {{persona_name}}

## 角色定位
...

## 核心能力
...

## 默认工具偏好
...

## 工作流程
...

## 输出格式
...

## 约束
...
```

## 4. 组合示例

```text
最终 Prompt = Base.md + Model/{{model_name}}.md + Persona/{{persona_name}}.md
```

示例：

```text
Base.md + Model/gpt-5.md + Persona/architect.md
```

## 5. 最佳实践

- 角色定义聚焦 1 个核心场景，避免“万能角色”
- 变量只保留必要项，过多变量会提升维护成本
- 新增模板后，至少做 3 轮真实对话回归测试
