# 20_pipeline_integration — 生产 Pipeline 集成

## 目标

把建模、材质、渲染、导出整合成可持续运行的生产 Pipeline。

## 前置条件

- 已具备基础脚本与批处理能力
- 有统一的项目目录与命名规范

## 操作步骤

1. 定义 Pipeline 阶段：`ingest -> build -> shade -> light -> render -> export`。
2. 为每个阶段定义输入、输出和失败处理策略。
3. 将阶段配置写入 YAML，并把环境差异外化到变量。
4. 在预发布环境跑一轮全链路验证。
5. 接入监控：耗时、失败率、重试次数、产物完整性。

## 验收标准

- 全链路可一键执行且可重复
- 任一阶段失败都能快速定位并重跑
- 产物结构稳定，满足交付规范

## 常见问题

- 隐式依赖导致失败：显式声明阶段输入输出
- 重跑成本高：实现阶段级缓存与断点续跑
