# 18_web_collaboration — Web 多人协作

## 目标

建立多人协作时的资产、版本与沟通规范，减少冲突。

## 前置条件

- 团队有统一仓库或共享存储
- 明确角色分工（建模/材质/灯光/动画）

## 操作步骤

1. 定义目录规范与命名规范（资产、贴图、缓存、导出）。
2. 建立分支/任务卡机制，避免多人修改同一资源。
3. 约定提交节奏（每日快照 + 里程碑版本）。
4. 使用渲染预览图做异步评审，减少口头误差。
5. 每周整理问题清单并更新团队模板。

## 验收标准

- 冲突率显著下降，回滚成本可控
- 任一资产都能追溯作者与版本
- 评审反馈可以映射到具体文件

## 常见问题

- 命名失控：建立自动校验脚本并在提交前执行
- 沟通成本高：统一评审模板与截图规范

## 可复用 Prompt

> 给我一份 Blender 项目多人协作规范，重点是版本与资产管理。
