bpy\_extras.view3d\_utils.region\_2d\_to\_vector\_3d(*region*, *rv3d*, *coord*)[¶](#bpy_extras.view3d_utils.region_2d_to_vector_3d "Link to this definition")
:   Return a direction vector from the viewport at the specific 2d region
    coordinate.

    Parameters:
    :   * **region** ([`bpy.types.Region`](../../bpy/types/Region.md#bpy.types.Region "bpy.types.Region")) – region of the 3D viewport, typically bpy.context.region.
        * **rv3d** ([`bpy.types.RegionView3D`](../../bpy/types/RegionView3D.md#bpy.types.RegionView3D "bpy.types.RegionView3D")) – 3D region data, typically bpy.context.space\_data.region\_3d.
        * **coord** (*2d vector*) – 2d coordinates relative to the region:
          (event.mouse\_region\_x, event.mouse\_region\_y) for example.

    Returns:
    :   normalized 3d vector.

    Return type:
    :   [`mathutils.Vector`](../../mathutils/index.md#mathutils.Vector "mathutils.Vector")

bpy\_extras.view3d\_utils.region\_2d\_to\_origin\_3d(*region*, *rv3d*, *coord*, *\**, *clamp=None*)[¶](#bpy_extras.view3d_utils.region_2d_to_origin_3d "Link to this definition")
:   Return the 3d view origin from the region relative 2d coords.

    Note

    Orthographic views have a less obvious origin,
    the far clip is used to define the viewport near/far extents.
    Since far clip can be a very large value,
    the result may give with numeric precision issues.

    To avoid this problem, you can optionally clamp the far clip to a
    smaller value based on the data you’re operating on.

    Parameters:
    :   * **region** ([`bpy.types.Region`](../../bpy/types/Region.md#bpy.types.Region "bpy.types.Region")) – region of the 3D viewport, typically bpy.context.region.
        * **rv3d** ([`bpy.types.RegionView3D`](../../bpy/types/RegionView3D.md#bpy.types.RegionView3D "bpy.types.RegionView3D")) – 3D region data, typically bpy.context.space\_data.region\_3d.
        * **coord** (*Sequence**[**float**]*) – 2D coordinates relative to the region;
          (event.mouse\_region\_x, event.mouse\_region\_y) for example.
        * **clamp** (*float* *|* *None*) – Clamp the maximum far-clip value used.
          (negative value will move the offset away from the view\_location)

    Returns:
    :   The origin of the viewpoint in 3d space.

    Return type:
    :   [`mathutils.Vector`](../../mathutils/index.md#mathutils.Vector "mathutils.Vector")

bpy\_extras.view3d\_utils.region\_2d\_to\_location\_3d(*region*, *rv3d*, *coord*, *depth\_location*)[¶](#bpy_extras.view3d_utils.region_2d_to_location_3d "Link to this definition")
:   Return a 3d location from the region relative 2d coords, aligned with
    *depth\_location*.

    Parameters:
    :   * **region** ([`bpy.types.Region`](../../bpy/types/Region.md#bpy.types.Region "bpy.types.Region")) – region of the 3D viewport, typically bpy.context.region.
        * **rv3d** ([`bpy.types.RegionView3D`](../../bpy/types/RegionView3D.md#bpy.types.RegionView3D "bpy.types.RegionView3D")) – 3D region data, typically bpy.context.space\_data.region\_3d.
        * **coord** (*2d vector*) – 2d coordinates relative to the region;
          (event.mouse\_region\_x, event.mouse\_region\_y) for example.
        * **depth\_location** (*3d vector*) – the returned vectors depth is aligned with this since
          there is no defined depth with a 2d region input.

    Returns:
    :   normalized 3d vector.

    Return type:
    :   [`mathutils.Vector`](../../mathutils/index.md#mathutils.Vector "mathutils.Vector")

bpy\_extras.view3d\_utils.location\_3d\_to\_region\_2d(*region*, *rv3d*, *coord*, *\**, *default=None*)[¶](#bpy_extras.view3d_utils.location_3d_to_region_2d "Link to this definition")
:   Return the *region* relative 2d location of a 3d position.

    Parameters:
    :   * **region** ([`bpy.types.Region`](../../bpy/types/Region.md#bpy.types.Region "bpy.types.Region")) – region of the 3D viewport, typically bpy.context.region.
        * **rv3d** ([`bpy.types.RegionView3D`](../../bpy/types/RegionView3D.md#bpy.types.RegionView3D "bpy.types.RegionView3D")) – 3D region data, typically bpy.context.space\_data.region\_3d.
        * **coord** (*3d vector*) – 3d world-space location.
        * **default** – Return this value if `coord`
          is behind the origin of a perspective view.

    Returns:
    :   2d location

    Return type:
    :   [`mathutils.Vector`](../../mathutils/index.md#mathutils.Vector "mathutils.Vector") | Any
