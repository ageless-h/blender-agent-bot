# SceneEEVEE(bpy\_struct)[¶](#sceneeevee-bpy-struct "Link to this heading")

base class — [`bpy_struct`](bpy_struct.md#bpy.types.bpy_struct "bpy.types.bpy_struct")

*class* bpy.types.SceneEEVEE(*bpy\_struct*)[¶](#bpy.types.SceneEEVEE "Link to this definition")
:   Scene display settings for 3D viewport

    bokeh\_max\_size[¶](#bpy.types.SceneEEVEE.bokeh_max_size "Link to this definition")
    :   Max size of the bokeh shape for the depth of field (lower is faster)

        Type:
        :   float in [0, 2000], default 100.0

    bokeh\_neighbor\_max[¶](#bpy.types.SceneEEVEE.bokeh_neighbor_max "Link to this definition")
    :   Maximum brightness to consider when rejecting bokeh sprites based on neighborhood (lower is faster)

        Type:
        :   float in [0, 100000], default 10.0

    bokeh\_overblur[¶](#bpy.types.SceneEEVEE.bokeh_overblur "Link to this definition")
    :   Apply blur to each jittered sample to reduce under-sampling artifacts

        Type:
        :   float in [0, 100], default 5.0

    bokeh\_threshold[¶](#bpy.types.SceneEEVEE.bokeh_threshold "Link to this definition")
    :   Brightness threshold for using sprite base depth of field

        Type:
        :   float in [0, 100000], default 1.0

    clamp\_surface\_direct[¶](#bpy.types.SceneEEVEE.clamp_surface_direct "Link to this definition")
    :   If non-zero, the maximum value for lights contribution on a surface. Higher values will be scaled down to avoid too much noise and slow convergence at the cost of accuracy. Used by light objects.

        Type:
        :   float in [0, inf], default 0.0

    clamp\_surface\_indirect[¶](#bpy.types.SceneEEVEE.clamp_surface_indirect "Link to this definition")
    :   If non-zero, the maximum value for indirect lighting on surface. Higher values will be scaled down to avoid too much noise and slow convergence at the cost of accuracy. Used by ray-tracing and light-probes.

        Type:
        :   float in [0, inf], default 10.0

    clamp\_volume\_direct[¶](#bpy.types.SceneEEVEE.clamp_volume_direct "Link to this definition")
    :   If non-zero, the maximum value for lights contribution in volumes. Higher values will be scaled down to avoid too much noise and slow convergence at the cost of accuracy. Used by light objects.

        Type:
        :   float in [0, inf], default 0.0

    clamp\_volume\_indirect[¶](#bpy.types.SceneEEVEE.clamp_volume_indirect "Link to this definition")
    :   If non-zero, the maximum value for indirect lighting in volumes. Higher values will be scaled down to avoid too much noise and slow convergence at the cost of accuracy. Used by light-probes.

        Type:
        :   float in [0, inf], default 0.0

    fast\_gi\_bias[¶](#bpy.types.SceneEEVEE.fast_gi_bias "Link to this definition")
    :   Bias the shading normal to reduce self intersection artifacts

        Type:
        :   float in [0, 1], default 0.05

    fast\_gi\_distance[¶](#bpy.types.SceneEEVEE.fast_gi_distance "Link to this definition")
    :   If non-zero, the maximum distance at which other surfaces will contribute to the fast GI approximation

        Type:
        :   float in [0, 100000], default 0.0

    fast\_gi\_method[¶](#bpy.types.SceneEEVEE.fast_gi_method "Link to this definition")
    :   Fast GI approximation method

        * `AMBIENT_OCCLUSION_ONLY`
          Ambient Occlusion – Use ambient occlusion instead of full global illumination.
        * `GLOBAL_ILLUMINATION`
          Global Illumination – Compute global illumination taking into account light bouncing off surrounding objects.

        Type:
        :   enum in [`'AMBIENT_OCCLUSION_ONLY'`, `'GLOBAL_ILLUMINATION'`], default `'GLOBAL_ILLUMINATION'`

    fast\_gi\_quality[¶](#bpy.types.SceneEEVEE.fast_gi_quality "Link to this definition")
    :   Precision of the fast GI ray marching

        Type:
        :   float in [0, 1], default 0.25

    fast\_gi\_ray\_count[¶](#bpy.types.SceneEEVEE.fast_gi_ray_count "Link to this definition")
    :   Amount of GI ray to trace for each pixel

        Type:
        :   int in [1, 16], default 2

    fast\_gi\_resolution[¶](#bpy.types.SceneEEVEE.fast_gi_resolution "Link to this definition")
    :   Control the quality of the fast GI lighting. Higher resolution uses more memory.

        * `1`
          1:1 – Full resolution.
        * `2`
          1:2 – Render this effect at 50% render resolution.
        * `4`
          1:4 – Render this effect at 25% render resolution.
        * `8`
          1:8 – Render this effect at 12.5% render resolution.
        * `16`
          1:16 – Render this effect at 6.25% render resolution.

        Type:
        :   enum in [`'1'`, `'2'`, `'4'`, `'8'`, `'16'`], default `'2'`

    fast\_gi\_step\_count[¶](#bpy.types.SceneEEVEE.fast_gi_step_count "Link to this definition")
    :   Amount of screen sample per GI ray

        Type:
        :   int in [1, 64], default 8

    fast\_gi\_thickness\_far[¶](#bpy.types.SceneEEVEE.fast_gi_thickness_far "Link to this definition")
    :   Angular thickness of the surfaces when computing fast GI and ambient occlusion. Reduces energy loss and missing occlusion of far geometry.

        Type:
        :   float in [0.0174533, 3.14159], default 0.785398

    fast\_gi\_thickness\_near[¶](#bpy.types.SceneEEVEE.fast_gi_thickness_near "Link to this definition")
    :   Geometric thickness of the surfaces when computing fast GI and ambient occlusion. Reduces light leaking and missing contact occlusion.

        Type:
        :   float in [0, 100000], default 0.25

    gi\_cubemap\_resolution[¶](#bpy.types.SceneEEVEE.gi_cubemap_resolution "Link to this definition")
    :   Size of every cubemaps

        Type:
        :   enum in [`'128'`, `'256'`, `'512'`, `'1024'`, `'2048'`, `'4096'`], default `'512'`

    gi\_diffuse\_bounces[¶](#bpy.types.SceneEEVEE.gi_diffuse_bounces "Link to this definition")
    :   Number of times the light is reinjected inside light grids, 0 disable indirect diffuse light

        Type:
        :   int in [0, inf], default 3

    gi\_glossy\_clamp[¶](#bpy.types.SceneEEVEE.gi_glossy_clamp "Link to this definition")
    :   Clamp pixel intensity to reduce noise inside glossy reflections from reflection cubemaps (0 to disable)

        Type:
        :   float in [0, inf], default 0.0

    gi\_irradiance\_pool\_size[¶](#bpy.types.SceneEEVEE.gi_irradiance_pool_size "Link to this definition")
    :   Size of the irradiance pool, a bigger pool size allows for more irradiance grid in the scene but might not fit into GPU memory and decrease performance

        Type:
        :   enum in [`'16'`, `'32'`, `'64'`, `'128'`, `'256'`, `'512'`, `'1024'`], default `'16'`

    gi\_visibility\_resolution[¶](#bpy.types.SceneEEVEE.gi_visibility_resolution "Link to this definition")
    :   Size of the shadow map applied to each irradiance sample

        Type:
        :   enum in [`'8'`, `'16'`, `'32'`, `'64'`], default `'32'`

    gtao\_distance[¶](#bpy.types.SceneEEVEE.gtao_distance "Link to this definition")
    :   Distance of object that contribute to the ambient occlusion effect

        Type:
        :   float in [0, 100000], default 0.2

    gtao\_quality[¶](#bpy.types.SceneEEVEE.gtao_quality "Link to this definition")
    :   Precision of the horizon search

        Type:
        :   float in [0, 1], default 0.25

    light\_threshold[¶](#bpy.types.SceneEEVEE.light_threshold "Link to this definition")
    :   Minimum light intensity for a light to contribute to the lighting

        Type:
        :   float in [0, inf], default 0.01

    motion\_blur\_depth\_scale[¶](#bpy.types.SceneEEVEE.motion_blur_depth_scale "Link to this definition")
    :   Lower values will reduce background bleeding onto foreground elements

        Type:
        :   float in [0, inf], default 100.0

    motion\_blur\_max[¶](#bpy.types.SceneEEVEE.motion_blur_max "Link to this definition")
    :   Maximum blur distance a pixel can spread over

        Type:
        :   int in [0, 2048], default 32

    motion\_blur\_steps[¶](#bpy.types.SceneEEVEE.motion_blur_steps "Link to this definition")
    :   Controls accuracy of motion blur, more steps means longer render time

        Type:
        :   int in [1, inf], default 1

    overscan\_size[¶](#bpy.types.SceneEEVEE.overscan_size "Link to this definition")
    :   Percentage of render size to add as overscan to the internal render buffers

        Type:
        :   float in [0, 50], default 3.0

    ray\_tracing\_method[¶](#bpy.types.SceneEEVEE.ray_tracing_method "Link to this definition")
    :   Select the tracing method used to find scene-ray intersections

        * `PROBE`
          Light Probe – Use light probes to find scene intersection.
        * `SCREEN`
          Screen-Trace – Raytrace against the depth buffer. Fallback to light probes for invalid rays..

        Type:
        :   enum in [`'PROBE'`, `'SCREEN'`], default `'SCREEN'`

    ray\_tracing\_options[¶](#bpy.types.SceneEEVEE.ray_tracing_options "Link to this definition")
    :   EEVEE settings for tracing reflections

        Type:
        :   [`RaytraceEEVEE`](RaytraceEEVEE.md#bpy.types.RaytraceEEVEE "bpy.types.RaytraceEEVEE"), (readonly)

    shadow\_pool\_size[¶](#bpy.types.SceneEEVEE.shadow_pool_size "Link to this definition")
    :   Size of the shadow pool, a bigger pool size allows for more shadows in the scene but might not fit into GPU memory

        Type:
        :   enum in [`'16'`, `'32'`, `'64'`, `'128'`, `'256'`, `'512'`, `'1024'`], default `'512'`

    shadow\_ray\_count[¶](#bpy.types.SceneEEVEE.shadow_ray_count "Link to this definition")
    :   Amount of shadow ray to trace for each light

        Type:
        :   int in [1, 4], default 1

    shadow\_resolution\_scale[¶](#bpy.types.SceneEEVEE.shadow_resolution_scale "Link to this definition")
    :   Resolution percentage of shadow maps

        Type:
        :   float in [0, 1], default 1.0

    shadow\_step\_count[¶](#bpy.types.SceneEEVEE.shadow_step_count "Link to this definition")
    :   Amount of shadow map sample per shadow ray

        Type:
        :   int in [1, 16], default 6

    taa\_render\_samples[¶](#bpy.types.SceneEEVEE.taa_render_samples "Link to this definition")
    :   Number of samples per pixel for rendering

        Type:
        :   int in [1, inf], default 64

    taa\_samples[¶](#bpy.types.SceneEEVEE.taa_samples "Link to this definition")
    :   Number of samples, unlimited if 0

        Type:
        :   int in [0, inf], default 16

    use\_bokeh\_jittered[¶](#bpy.types.SceneEEVEE.use_bokeh_jittered "Link to this definition")
    :   Jitter camera position to create accurate blurring using render samples (only for final render)

        Type:
        :   boolean, default False

    use\_fast\_gi[¶](#bpy.types.SceneEEVEE.use_fast_gi "Link to this definition")
    :   Use faster global illumination technique for high roughness surfaces

        Type:
        :   boolean, default False

    use\_gtao[¶](#bpy.types.SceneEEVEE.use_gtao "Link to this definition")
    :   Enable ambient occlusion to simulate medium scale indirect shadowing

        Type:
        :   boolean, default False

    use\_overscan[¶](#bpy.types.SceneEEVEE.use_overscan "Link to this definition")
    :   Internally render past the image border to avoid screen-space effects disappearing

        Type:
        :   boolean, default False

    use\_raytracing[¶](#bpy.types.SceneEEVEE.use_raytracing "Link to this definition")
    :   Enable the ray-tracing module

        Type:
        :   boolean, default False

    use\_shadow\_jitter\_viewport[¶](#bpy.types.SceneEEVEE.use_shadow_jitter_viewport "Link to this definition")
    :   Enable jittered shadows on the viewport. (Jittered shadows are always enabled for final renders).

        Type:
        :   boolean, default False

    use\_shadows[¶](#bpy.types.SceneEEVEE.use_shadows "Link to this definition")
    :   Enable shadow casting from lights

        Type:
        :   boolean, default False

    use\_taa\_reprojection[¶](#bpy.types.SceneEEVEE.use_taa_reprojection "Link to this definition")
    :   Denoise image using temporal reprojection (can leave some ghosting)

        Type:
        :   boolean, default True

    use\_volume\_custom\_range[¶](#bpy.types.SceneEEVEE.use_volume_custom_range "Link to this definition")
    :   Enable custom start and end clip distances for volume computation

        Type:
        :   boolean, default False

    use\_volumetric\_shadows[¶](#bpy.types.SceneEEVEE.use_volumetric_shadows "Link to this definition")
    :   Cast shadows from volumetric materials onto volumetric materials (Very expensive)

        Type:
        :   boolean, default False

    volumetric\_end[¶](#bpy.types.SceneEEVEE.volumetric_end "Link to this definition")
    :   End distance of the volumetric effect

        Type:
        :   float in [1e-06, inf], default 100.0

    volumetric\_light\_clamp[¶](#bpy.types.SceneEEVEE.volumetric_light_clamp "Link to this definition")
    :   Maximum light contribution, reducing noise

        Type:
        :   float in [0, inf], default 0.0

    volumetric\_ray\_depth[¶](#bpy.types.SceneEEVEE.volumetric_ray_depth "Link to this definition")
    :   Maximum surface intersection count used by the accurate volume intersection method. Will create artifact if it is exceeded. Higher count increases VRAM usage.

        Type:
        :   int in [1, 16], default 16

    volumetric\_sample\_distribution[¶](#bpy.types.SceneEEVEE.volumetric_sample_distribution "Link to this definition")
    :   Distribute more samples closer to the camera

        Type:
        :   float in [0, 1], default 0.8

    volumetric\_samples[¶](#bpy.types.SceneEEVEE.volumetric_samples "Link to this definition")
    :   Number of steps to compute volumetric effects. Higher step count increase VRAM usage and quality.

        Type:
        :   int in [1, 256], default 64

    volumetric\_shadow\_samples[¶](#bpy.types.SceneEEVEE.volumetric_shadow_samples "Link to this definition")
    :   Number of samples to compute volumetric shadowing

        Type:
        :   int in [1, 128], default 16

    volumetric\_start[¶](#bpy.types.SceneEEVEE.volumetric_start "Link to this definition")
    :   Start distance of the volumetric effect

        Type:
        :   float in [1e-06, inf], default 0.1

    volumetric\_tile\_size[¶](#bpy.types.SceneEEVEE.volumetric_tile_size "Link to this definition")
    :   Control the quality of the volumetric effects. Higher resolution uses more memory.

        * `1`
          1:1 – Full resolution.
        * `2`
          1:2 – Render this effect at 50% render resolution.
        * `4`
          1:4 – Render this effect at 25% render resolution.
        * `8`
          1:8 – Render this effect at 12.5% render resolution.
        * `16`
          1:16 – Render this effect at 6.25% render resolution.

        Type:
        :   enum in [`'1'`, `'2'`, `'4'`, `'8'`, `'16'`], default `'8'`
