:   Load a FBX file

    Parameters:
    :   * **filepath** (*string**,* *(**optional**,* *never None**)*) ‚Äì File Path, Filepath used for importing the file
        * **directory** (*string**,* *(**optional**,* *never None**)*) ‚Äì directory
        * **filter\_glob** (*string**,* *(**optional**,* *never None**)*) ‚Äì filter\_glob
        * **files** (`bpy_prop_collection` of `OperatorFileListElement`, (optional)) ‚Äì File Path
        * **ui\_tab** (enum in [`'MAIN'`, `'ARMATURE'`], (optional)) ‚Äì

          ui\_tab, Import options categories

          + `MAIN`
            Main ‚Äì Main basic settings.
          + `ARMATURE`
            Armatures ‚Äì Armature-related settings.
        * **use\_manual\_orientation** (*boolean**,* *(**optional**)*) ‚Äì Manual Orientation, Specify orientation and scale, instead of using embedded data in FBX file
        * **global\_scale** (*float in* *[**0.001**,* *1000**]**,* *(**optional**)*) ‚Äì Scale
        * **bake\_space\_transform** (*boolean**,* *(**optional**)*) ‚Äì Apply Transform, Bake space transform into object data, avoids getting unwanted rotations to objects when target space is not aligned with Blender‚Äôs space (WARNING! experimental option, use at own risk, known to be broken with armatures/animations)
        * **use\_custom\_normals** (*boolean**,* *(**optional**)*) ‚Äì Custom Normals, Import custom normals, if available (otherwise Blender will recompute them)
        * **colors\_type** (enum in [`'NONE'`, `'SRGB'`, `'LINEAR'`], (optional)) ‚Äì

          Vertex Colors, Import vertex color attributes

          + `NONE`
            None ‚Äì Do not import color attributes.
          + `SRGB`
            sRGB ‚Äì Expect file colors in sRGB color space.
          + `LINEAR`
            Linear ‚Äì Expect file colors in linear color space.
        * **use\_image\_search** (*boolean**,* *(**optional**)*) ‚Äì Image Search, Search subdirs for any associated images (WARNING: may be slow)
        * **use\_alpha\_decals** (*boolean**,* *(**optional**)*) ‚Äì Alpha Decals, Treat materials with alpha as decals (no shadow casting)
        * **decal\_offset** (*float in* *[**0**,* *1**]**,* *(**optional**)*) ‚Äì Decal Offset, Displace geometry of alpha meshes
        * **use\_anim** (*boolean**,* *(**optional**)*) ‚Äì Import Animation, Import FBX animation
        * **anim\_offset** (*float in* *[**-inf**,* *inf**]**,* *(**optional**)*) ‚Äì Animation Offset, Offset to apply to animation during import, in frames
        * **use\_subsurf** (*boolean**,* *(**optional**)*) ‚Äì Subdivision Data, Import FBX subdivision information as subdivision surface modifiers
        * **use\_custom\_props** (*boolean**,* *(**optional**)*) ‚Äì Custom Properties, Import user properties as custom properties
        * **use\_custom\_props\_enum\_as\_string** (*boolean**,* *(**optional**)*) ‚Äì Import Enums As Strings, Store enumeration values as strings
        * **ignore\_leaf\_bones** (*boolean**,* *(**optional**)*) ‚Äì Ignore Leaf Bones, Ignore the last bone at the end of each chain (used to mark the length of the previous bone)
        * **force\_connect\_children** (*boolean**,* *(**optional**)*) ‚Äì Force Connect Children, Force connection of children bones to their parent, even if their computed head/tail positions do not match (can be useful with pure-joints-type armatures)
        * **automatic\_bone\_orientation** (*boolean**,* *(**optional**)*) ‚Äì Automatic Bone Orientation, Try to align the major bone axis with the bone children
        * **primary\_bone\_axis** (enum in [`'X'`, `'Y'`, `'Z'`, `'-X'`, `'-Y'`, `'-Z'`], (optional)) ‚Äì Primary Bone Axis
        * **secondary\_bone\_axis** (enum in [`'X'`, `'Y'`, `'Z'`, `'-X'`, `'-Y'`, `'-Z'`], (optional)) ‚Äì Secondary Bone Axis
        * **use\_prepost\_rot** (*boolean**,* *(**optional**)*) ‚Äì Use Pre/Post Rotation, Use pre/post rotation from FBX transform (you may have to disable that in some cases)
        * **mtl\_name\_collision\_mode** (enum in [`'MAKE_UNIQUE'`, `'REFERENCE_EXISTING'`], (optional)) ‚Äì

          Material Name Collision, Behavior when the name of an imported material conflicts with an existing material

          + `MAKE_UNIQUE`
            Make Unique ‚Äì Import each FBX material as a unique Blender material.
          + `REFERENCE_EXISTING`
            Reference Existing ‚Äì If a material with the same name already exists, reference that instead of importing.
        * **axis\_forward** (enum in [`'X'`, `'Y'`, `'Z'`, `'-X'`, `'-Y'`, `'-Z'`], (optional)) ‚Äì Forward
        * **axis\_up** (enum in [`'X'`, `'Y'`, `'Z'`, `'-X'`, `'-Y'`, `'-Z'`], (optional)) ‚Äì Up

bpy.ops.import\_scene.gltf(*\**, *filepath=''*, *export\_import\_convert\_lighting\_mode='SPEC'*, *filter\_glob='\*.glb;\*.gltf'*, *directory=''*, *files=None*, *loglevel=0*, *import\_pack\_images=True*, *merge\_vertices=False*, *import\_shading='NORMALS'*, *bone\_heuristic='BLENDER'*, *disable\_bone\_shape=False*, *bone\_shape\_scale\_factor=1.0*, *guess\_original\_bind\_pose=True*, *import\_webp\_texture=False*, *import\_unused\_materials=False*, *import\_select\_created\_objects=True*, *import\_scene\_extras=True*, *import\_scene\_as\_collection=True*, *import\_merge\_material\_slots=True*)[¬∂](#bpy.ops.import_scene.gltf "Link to this definition")
:   Load a glTF 2.0 file

    Parameters:
    :   * **filepath** (*string**,* *(**optional**,* *never None**)*) ‚Äì File Path, Filepath used for importing the file
        * **export\_import\_convert\_lighting\_mode** (enum in [`'SPEC'`, `'COMPAT'`, `'RAW'`], (optional)) ‚Äì

          Lighting Mode, Optional backwards compatibility for non-standard render engines. Applies to lights

          + `SPEC`
            Standard ‚Äì Physically-based glTF lighting units (cd, lx, nt).
          + `COMPAT`
            Unitless ‚Äì Non-physical, unitless lighting. Useful when exposure controls are not available.
          + `RAW`
            Raw (Deprecated) ‚Äì Blender lighting strengths with no conversion.
        * **filter\_glob** (*string**,* *(**optional**,* *never None**)*) ‚Äì filter\_glob
        * **directory** (*string**,* *(**optional**,* *never None**)*) ‚Äì directory
        * **files** (`bpy_prop_collection` of `OperatorFileListElement`, (optional)) ‚Äì File Path
        * **loglevel** (*int in* *[**-inf**,* *inf**]**,* *(**optional**)*) ‚Äì Log Level, Log Level
        * **import\_pack\_images** (*boolean**,* *(**optional**)*) ‚Äì Pack Images, Pack all images into .blend file
        * **merge\_vertices** (*boolean**,* *(**optional**)*) ‚Äì Merge Vertices, The glTF format requires discontinuous normals, UVs, and other vertex attributes to be stored as separate vertices, as required for rendering on typical graphics hardware. This option attempts to combine co-located vertices where possible. Currently cannot combine verts with different normals
        * **import\_shading** (enum in [`'NORMALS'`, `'FLAT'`, `'SMOOTH'`], (optional)) ‚Äì Shading, How normals are computed during import
        * **bone\_heuristic** (enum in [`'BLENDER'`, `'TEMPERANCE'`, `'FORTUNE'`], (optional)) ‚Äì

          Bone Dir, Heuristic for placing bones. Tries to make bones pretty

          + `BLENDER`
            Blender (best for import/export round trip) ‚Äì Good for re-importing glTFs exported from Blender, and re-exporting glTFs to glTFs after Blender editing. Bone tips are placed on their local +Y axis (in glTF space).
          + `TEMPERANCE`
            Temperance (average) ‚Äì Decent all-around strategy. A bone with one child has its tip placed on the local axis closest to its child.
          + `FORTUNE`
            Fortune (may look better, less accurate) ‚Äì Might look better than Temperance, but also might have errors. A bone with one child has its tip placed at its child‚Äôs root. Non-uniform scalings may get messed up though, so beware.
        * **disable\_bone\_shape** (*boolean**,* *(**optional**)*) ‚Äì Disable Bone Shape, Do not create bone shapes
        * **bone\_shape\_scale\_factor** (*float in* *[**-inf**,* *inf**]**,* *(**optional**)*) ‚Äì Bone Shape Scale, Scale factor for bone shapes
        * **guess\_original\_bind\_pose** (*boolean**,* *(**optional**)*) ‚Äì Guess Original Bind Pose, Try to guess the original bind pose for skinned meshes from the inverse bind matrices. When off, use default/rest pose as bind pose
        * **import\_webp\_texture** (*boolean**,* *(**optional**)*) ‚Äì Import WebP Textures, If a texture exists in WebP format, loads the WebP texture instead of the fallback PNG/JPEG one
        * **import\_unused\_materials** (*boolean**,* *(**optional**)*) ‚Äì Import Unused Materials & Images, Import materials & Images not assigned to any mesh
        * **import\_select\_created\_objects** (*boolean**,* *(**optional**)*) ‚Äì Select Imported Objects, Select created objects at the end of the import
        * **import\_scene\_extras** (*boolean**,* *(**optional**)*) ‚Äì Import Scene Extras, Import scene extras as custom properties. Existing custom properties will be overwritten
        * **import\_scene\_as\_collection** (*boolean**,* *(**optional**)*) ‚Äì Import Scene as Collection, Import the scene as a collection
        * **import\_merge\_material\_slots** (*boolean**,* *(**optional**)*) ‚Äì Merge Material Slot when possible, Merge material slots when possible
